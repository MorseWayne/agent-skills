# Concept Note Guide

Use concept notes when a term is important, obscure, overloaded, or cross-cutting.

## Glossary Entry

```markdown
## <Term> / <中文名>

一句话：<plain explanation>

在本需求中：<feature-specific explanation>

相关位置：
- <path>

容易误解：
- <misunderstanding>
```

## Deep Concept Note

```markdown
# <Concept> / <中文名>

## 一句话理解

## 它解决什么问题

## 为什么这个需求里需要它

## 在代码里的体现

## 前端视角

## 后端视角

## 架构视角

## 如果不理解它，会出什么 bug

## 和相近概念的区别

## 自测题
```

## Examples of Concepts Worth Explaining

- API Contract / API 契约
- Idempotency / 幂等性
- Optimistic Update / 乐观更新
- Cache Invalidation / 缓存失效
- Transaction / 事务
- Eventual Consistency / 最终一致性
- Dependency Injection / 依赖注入
- Adapter / 适配器
- Invariant / 不变量
- DTO / 数据传输对象
- E2E Test / 端到端测试
